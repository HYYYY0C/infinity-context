## 💡 工作原理

### **1. 智能监控（每 5 分钟）**

- 检查会话使用率（>80% 触发）
- 检测自然断点（"好的"、"谢谢"等）
- 后台预生成摘要

### **2. 自动执行**

检测到合适时机 → 自动完成：
```python
1. 生成结构化摘要（LLM）
2. 保存摘要（terminal）
3. 等待自然断点
4. Computer Use 执行 /new
5. 检测新会话开启
6. 自动注入摘要
7. 事后通知用户
```

### **3. 用户视角**

**完全无感知！**
- ❌ 不需要点击任何按钮
- ❌ 不需要手动执行 `/new`
- ❌ 不需要说"继续"
- ✅ 继续对话即可

## 🔧 技术亮点

### **1. Computer Use 自动执行**

```python
def auto_execute_new():
    # 聚焦输入框
    computer_use.click_element("输入框")
    
    # 输入 /new
    computer_use.type_text("/new")
    
    # 按 Enter
    computer_use.press_key("Enter")
```

### **2. 自然断点检测**

```python
natural_breaks = [
    "好的", "明白了", "谢谢", "理解了",
    "ok", "good", "thanks",
    "👌", "✅", "🙏"
]

# 检测到自然断点才执行切换
if is_natural_break(last_message):
    execute_auto_switch()
```

### **3. 事后通知**

```python
# 切换完成后，温和通知用户
send_message("""
✅ 已自动切换到新会话

为了保持最佳性能，我已自动：
1. 生成了上一会话的摘要
2. 开启了新会话
3. 注入了上下文

我们可以继续了！
""")
```
