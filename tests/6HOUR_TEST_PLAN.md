# 🧪 Hermes Session Rotator - 6 小时实际测试计划

**测试开始时间**: 2026-07-14  
**测试时长**: 6 小时  
**测试目标**: 验证核心功能在实际工作场景中的稳定性

---

## 📋 测试场景

### 场景 A: 日常对话管理

**测试内容**:
- 正常使用 Hermes 处理各种任务
- 每 2-3 小时或对话超过 40 轮时，使用 `/compress`
- 观察摘要质量和切换流畅度

**验证点**:
- [ ] 摘要生成速度 <3 秒
- [ ] 摘要字数在 300-500 之间
- [ ] 包含所有核心任务
- [ ] 待办事项完整
- [ ] 新 session 能理解上下文

---

### 场景 B: 长时间研究任务

**测试内容**:
- 选择一个复杂主题（如：研究 Hermes 最新 PR）
- 进行深度研究（预计 2-3 小时）
- 使用 `session-rotator` 管理对话

**验证点**:
- [ ] 能处理 60+ 轮对话
- [ ] 多次切换后上下文不丢失
- [ ] 摘要能准确反映研究进度
- [ ] 待办事项跟踪准确

---

### 场景 C: 多项目并行

**测试内容**:
- 同时处理 2-3 个项目
- 每个项目使用独立的 session
- 使用摘要快速切换上下文

**验证点**:
- [ ] 不同项目上下文不混淆
- [ ] 摘要能准确区分项目
- [ ] 切换后能快速进入状态
